# 策略因子探索实验报告

日期:2026-07-18
状态:实验完成 —— **结论:当前不采纳任何因子变体,基线保持默认**

## TL;DR(诚实结论)

针对基线策略的弱点(回撤大、夏普低),新增了 3 个候选因子(波动率过滤、趋势质量、相对强度)并在**两个独立时间窗**上回测对比。结果:**没有任何变体在两个窗口都稳健改善**。看似最优的组合其实是过拟合到某一个窗口。**建议保持基线(趋势 0.4 / 动量 0.4 / 量能 0.2)为默认**,不采纳任何变体。这是一个诚实的负面结果——它的价值在于用客观数据否定了"加因子就能改善"的直觉,并暴露了过拟合风险。

## 实验设计

- 候选因子(纯函数,已加单测,**未接入 default_screener**):
  - `VolatilityRule`(波动率过滤,低波动加分,意在降回撤)
  - `TrendQualityRule`(趋势平滑度,震荡上涨扣分)
  - `RelativeStrengthRule`(相对 SPY 基准的强弱)
- 两个时间窗(同配置:10 万本金、最多 5 持仓、min_score 0.5、5bp 滑点、30 只默认股票池):
  - **P1 基线窗** 2024-07-01 → 2025-06-30(基线偏弱的一年)
  - **P2 稳健性窗** 2023-07-01 → 2024-06-30(基线很强的一年,牛市)
- 每窗数据抓一次复用,跑 baseline + 各因子单独 + 两个组合,对比 收益/回撤/夏普/胜率/成交数。

## 两窗口对比

**P1(2024H2–2025H1)——基线偏弱:**

| 变体 | 总收益 | 最大回撤 | 夏普 | 胜率 |
|---|---|---|---|---|
| **baseline** (t.4/m.4/v.2) | 3.28% | −18.57% | 0.261 | 48.96% |
| +volatility | −2.04% | −15.45% | −0.064 | 46.12% |
| +trend_quality | −5.29% | −18.77% | −0.221 | 46.05% |
| +relative_strength | 5.94% | −16.26% | **0.381** | 46.00% |
| combined_defensive | 0.10% | −14.49% | 0.080 | 44.71% |
| combined_all | 5.96% | **−12.08%** | **0.467** | 46.64% |

**P2(2023H2–2024H1)——基线很强(牛市):**

| 变体 | 总收益 | 最大回撤 | 夏普 | 胜率 |
|---|---|---|---|---|
| **baseline** (t.4/m.4/v.2) | 30.31% | −21.28% | **1.451** | 50.00% |
| +volatility | 17.35% | **−12.17%** | 1.273 | 50.53% |
| +trend_quality | 6.06% | −14.39% | 0.497 | 45.08% |
| +relative_strength | 21.20% | −23.51% | 1.004 | 49.40% |
| combined_defensive | 2.18% | −11.04% | 0.247 | 46.18% |
| combined_all | 13.39% | −15.76% | 0.932 | 50.00% |

## 逐因子分析

- **relative_strength**:P1 的最佳单因子(夏普 0.26→0.38),但 P2 反而变差(夏普 1.45→1.00,回撤 −21%→−23.5% 更糟)。**窗口依赖,不稳健。**
- **volatility filter**:两个窗口**都降回撤**(P1 −18.6%→−15.5%,P2 −21.3%→−12.2% 大降)——这是全实验里唯一稳健的效果。但代价是**系统性削收益**(P1 转负,P2 从 30% 削到 17%),夏普并未改善(P1 转负,P2 略降)。**是一个"低回撤 vs 收益"的权衡工具,不是免费的改进。**
- **trend_quality**:两个窗口**都差**(收益和夏普都被削)。**应弃。**
- **combined_all**:在 P1 看着最优(夏普 0.26→0.47、回撤 −18.6%→−12%),**但这恰恰是过拟合信号**——它在 P2 牛市把收益从 30% 削到 13%、夏普从 1.45 削到 0.93。它对 P1 这个特定弱窗"优化"了,靠牺牲牛市上行换取弱市的好看数字。

## 过拟合警示

在弱年(P1)看着最好的两个变体(combined_all、relative_strength),在强年(P2)都显著跑输基线。这是过拟合的教科书表现:**只在一个窗口选"最优",几乎必然选中一个恰好契合那段行情的配置。** 若只跑 P1、不做 P2 稳健性检验,就会误以为 combined_all 是明确改进——两窗对比正是为了防这个陷阱。

另一个诚实观察:基线在 P2 其实很强(30% / 夏普 1.45)。P1 的弱主要是市场环境(那段震荡市),不是基线"坏"。基线是"随市场"的,加因子想在弱年补,代价是牛年跑输。

## 沉淀与建议

- **默认策略保持不变**(基线 t.4/m.4/v.2)。不采纳任何变体。
- 3 个候选因子(有单测、未接入默认)+ 实验框架 `scripts/strategy_experiment.py` 保留在代码库,作为**未来 A/B 的可复用资产**。想测新因子时改 `build_variants()` 再跑即可。
- 若用户明确要一个**"低回撤优先"**的可选配置(愿意牺牲收益换平稳),volatility-tilted 变体是唯一有稳健降回撤证据的候选——但必须标注清楚"削收益"的权衡,且不作默认。
- 真正提升风险调整后收益,可能需要的不是"再加一个技术因子",而是:更好的仓位管理/止损、更大更优的股票池、或 M2 的 LLM 委员会在基本面/事件层面的增益——这些是更大的方向,留待后续。
