# 市场状态(regime)择时实验报告

日期:2026-07-19
状态:实验完成 —— **结论:SPY-200日均线 regime 过滤器不是稳健的免费改进;强烈依赖市场形态,不采纳为默认**

## TL;DR(诚实结论)

给基线策略叠加一个大盘 regime 过滤器(SPY 跌破 200 日均线 → 转防御:清仓/不新开,回到均线上方再恢复),在**四个不同形态的时间窗**上回测。结果:**它在"缓慢磨人的下跌/震荡市"里确实有效,但在"V 型急跌急涨"里灾难性失效**——不仅收益被腰斩,回撤反而更深(卖在底部、错过反弹)。这是趋势/均线择时的经典软肋。

因此:**这是本项目第三个诚实实验,结论与前两个(因子、止损)一致——简单技术叠加都强依赖市场形态、没有免费的午餐。** regime 过滤器不作默认;它是一个"你判断未来是慢熊、且愿意承担 V 型 whipsaw 风险"时的**可选**风险偏好工具,且必须公开披露其 2020 型失效。真正可能更根本的改进方向仍是:更低相关/更大的股票池分散、以及 M2 LLM 委员会在基本面/事件层面的增益(后者的历史 alpha 验证受新闻数据缺口所限)。

## 实验设计

- 变体(同一 `default_screener` 趋势.4/动量.4/量能.2,只改 regime 闸门):
  - `baseline`:无 regime 过滤(即现有 `BacktestEngine`)。
  - `regime_flat`:SPY 收盘 < 其 200 日均线 → **清仓并持现金**,不新开;回到均线上恢复。
  - `regime_no_new`:SPY < 200SMA → **只是不新开**(保留存量,靠正常轮换退出)。
- 关键防未来函数:regime 信号 as-of 某日只用 SPY ≤ 该日的收盘算均线(单测 `test_lookahead_safe_ignores_future` 双向验证:as_of 之后无论暴跌或暴涨都不改变当日判定)。regime 引擎在 `regime_mode=None` 时与基线逐比特等价(单测 `test_regime_none_matches_baseline_engine`,非空成交下 equity 曲线/成交/指标全等)。SPY 仅作信号,不进交易池。
- 四个时间窗(同配置:10 万本金、最多 5 持仓、min_score 0.5、5bp 滑点、250 回看、30 只默认池;SPY 额外多取 400 天以保证窗首均线已定义)。**故意各选一种形态**,同一套参数(200SMA 未调优)跑全部窗口以防过拟合。

## 逐窗口分析

- **强牛(2023)**:过滤器几乎不动(risk-off 仅 2%),小幅削收益(30.3→27.2)、小幅降回撤(−21.3→−19.3),夏普略降。代价小、收益也小——基本中性偏微负。
- **震荡弱市(2024H2)**:`regime_flat` **明显改善**——收益 3.3→12.6、回撤 −18.6→−16.7、夏普 0.26→0.76。它在这段避开了几波下跌。
- **磨人真熊(2022)**:`regime_flat` **有效保本**——亏损从 −27.6% 收窄到 −15.3%,最大回撤从 −37.4% **腰斩到 −15.3%**。(注:此窗夏普反而更差 −1.39→−2.22,是"大部分时间持现金→曲线近乎零波动但仍微亏→夏普=均值/标准差被放大为大负"的度量假象;熊市里应看收益/回撤这两个保本指标,`regime_flat` 明显赢。)
- **V 型急跌急涨(2020)——对抗性考场,决定性结果**:`regime_flat` **灾难性失效**——收益 24.8%→**2.8%**,而且回撤不降反升 −25.8%→**−33.8%**。原因是 200SMA 太慢:急跌太快,均线直到跌了一大截才触发防御,于是**卖在接近底部**(把浮亏做实),随后持现金**错过急速反弹**,等价格重新站上均线才追回来——典型 whipsaw:卖低、踏空。回撤反而更深,正因为它在底部实现了亏损、又没吃到本可修复回撤的反弹。

## 稳健性/过拟合评估(诚实)

- 这**不是**过拟合(200SMA 未调参,同一规则跑全部窗;它在 2022 与 2024H2 两个不同的弱窗都帮上忙,机制也经济上自洽),但它是**强形态依赖**:对"慢熊/震荡"有效,对"V 型急跌急涨"有害。
- 核心张力不可消除:均线慢 → 急跌反应太迟、在底部 whipsaw;若把均线调快(如 50 日)→ 震荡市里 whipsaw 更频繁。快慢都各有失效场景,没有一个设置在所有形态都赢。
- 只跑到 2020/2022 各一次极端窗,样本仍少;更多熊市(2008、2018Q4)与更多 V 型会进一步刻画分布,但 2020 这一个对抗窗已足以否定"regime 过滤是稳健免费改进"的结论。
- 这本质是**降低下行期的市场 beta 敞口的择时叠加**,不是选股 alpha;是否净正取决于未来是慢熊还是 V 型——事前不可知。

## 沉淀与建议

- **默认策略保持不变**(基线,无 regime 过滤)。不采纳为默认。
- 若用户明确判断"未来是磨人的慢熊/震荡、且愿意承担 V 型 whipsaw 风险",`regime_flat` 是唯一在慢熊/震荡两窗都有稳健保本证据的可选叠加——但必须标注清楚它在 2020 型行情里"收益腰斩且回撤更深"的失效,且不作默认。`regime_no_new` 全面弱于 `regime_flat`,不推荐。
- 资产沉淀(有单测、防未来函数、`regime_mode=None` 等价基线,均**未接入默认**):`app/backtest/regime_signal.py`、`app/backtest/regime_engine.py`、实验框架 `scripts/regime_experiment.py`、本报告。想测别的 regime 规则(双均线/波动率/回撤触发)时改 `REGIME_VARIANTS` 再跑即可。
- **三轮技术叠加实验(因子 / 止损 / regime 择时)一致的元结论**:简单技术叠加都强依赖市场形态、没有稳健的免费改进;基线本质是随市场的 beta,每个叠加只是拿一种行情的结果换另一种行情的结果。真正不同性质的杠杆仍是:①更大/更低相关的股票池(分散度,纯量化可回测,尚未做)②M2 LLM 委员会的基本面/事件增益(历史 alpha 验证受新闻数据缺口所限)。
